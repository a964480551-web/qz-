## 一、功能架构

```
┌─────────────────────────────────────────┐
│           搜索推荐词流程                 │
├─────────────────────────────────────────┤
│                                         │
│   用户搜索记录 ──→ 数据清洗 ──→ 推荐词库 │
│        ↓                                │
```

## 二、推荐词展示格式

### 2.1 格式定义

```
[素材名称] + [分类]

示例：
├── 现代简约客厅
├── 北欧风格卧室
├── 新中式书房
└── 轻奢风餐厅
```

### 2.2 数据结构

| 字段 | 类型 | 说明 |
|------|------|------|
| 素材名称 | String | 模型/素材的原始名称 |
| 分类 | String | 所属风格/空间分类 |
| 创建时间 | DateTime | 素材上传时间（排序依据） |
| 拆词得分 | Integer | 匹配度评分（综合排序用） |

## 三、数据来源与清洗

### 3.1 数据来源分布

```
用户搜索记录 100%
├── 正常搜索词 ~85%
├── 测试数据 ~10%  ──→ 已过滤
└── 异常/不规范 ~5%  ──→ 需手动清理
```

### 3.2 清洗机制

| 清洗层级 | 规则 | 状态 |
|----------|------|------|
| **第一层：自动过滤** | 包含"测试"字样 | ✅ 已实施 |
| **第二层：人工审核** | 明显无关词汇（如"日式"、"卧室"） | ⚠️ 需权限 |
| **第三层：定期维护** | 后台手动删除异常素材名称 | ⚠️ 需权限 |

### 3.3 清洗流程

```
数据入库
   ↓
[过滤测试词汇] ──→ 自动剔除含"测试"关键词
   ↓
[异常词检测] ──→ 人工标记无关词汇
   ↓
[后台删除] ──→ 需管理员权限执行
   ↓
[进入推荐库]
```

## 五、排序规则

### 5.1 排序方式对比

| 维度 | 最新排序 | 综合排序 |
|------|----------|----------|
| **排序依据** | 创建时间倒序 | 拆词得分 + 序号 |
| **适用场景** | 展示新素材 | 展示高质量素材 |
| **存在问题** | 可能遗漏高质旧素材 | 老旧内容长期占据前列 |
| **默认状态** | ✅ 当前默认 | 需手动切换 |

### 5.2 排序规则演变

```
时间线
───────────────────────────────────────────────►

2025/3/18  ┐
           │  【原规则】综合排序
           │  • 先按拆词得分
           │  • 再按序号/时间
           │
           │  ⚠️ 问题：得分90分的老旧内容
           │     永远排在得分89分的新内容前
           │
2025/3/23  ┤
           │  【优化方案】默认改为"最新"
           │  • 搜索默认按时间倒序
           │  • 用户可手动切回"综合"
           │
2025/3/25  ┤
           │  【妥协方案】全局生效
           │  • 搜索+目录统一改为"最新"
           │  • 原因：UI组件共用，改动困难
           │
2025/4/9   ┘
           【Bug】最新与综合内容一致
           原因：前端sortType参数未传递
```

### 5.3 当前排序逻辑

```
默认展示：
┌─────────────┐
│   最新      │ ← 按创建时间倒序
│  （默认）   │
└─────────────┘
      ↓ 点击切换
┌─────────────┐
│   综合      │ ← 按拆词得分排序
│  （旧规则） │
└─────────────┘
```

```
        ┌─────────────────┐
        │    搜索模块     │
        └────────┬────────┘
                 │
        ┌────────▼────────┐

                 │
    ┌────────────┼────────────┐
    ↓            ↓            ↓
 搜索页       目录页       替换页
```

### 6.2 技术债务清单

| 问题 | 影响 | 优先级 |
|------|------|--------|
| 搜索与目录共用UI组件 | 改搜索影响目录排序 | P0 |

| 改动类型 | 难度 | 原因 |
|----------|------|------|
| 独立搜索排序 | 高 | 代码耦合严重 |
| 修复参数传递 | 中 | 需定位传参逻辑 |
