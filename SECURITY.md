# 安全说明

## 报告问题

请不要在公开 Issue 中提交有效账号、密码、Token、数据库内容或用户资料。发现安全问题时，请通过仓库维护者提供的私密联系方式报告。

## 密钥管理

- 所有密钥和初始化凭据必须通过环境变量提供。
- 不要把 `.env`、服务器日志、数据库或上传目录提交到版本控制。
- 如果密钥曾进入公开 Git 历史，请立即轮换；仅删除文件或改为私有仓库不能撤销泄露。
- 生产环境应使用独立、随机且长度足够的 JWT 密钥和管理员密码。

## 数据处理

管理员密码仅以 bcrypt 哈希形式存储。部署者仍需自行落实 HTTPS、访问控制、数据库备份加密、日志脱敏和最小权限原则。
